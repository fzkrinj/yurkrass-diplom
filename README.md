## Проект «Юркрас» — сайт юридической компании с онлайн‑консультантом

Веб‑приложение для юридической компании «Юркрас». Сайт предоставляет информацию об услугах, позволяет оставлять заявки на консультацию, общаться с онлайн‑консультантом (чат‑бот + подключение живого консультанта), а также содержит личный кабинет клиента и кабинет юриста.

Проект ориентирован на запуск локально под XAMPP (Apache + PHP + MySQL).

### Стек

- **Фронтенд**: HTML, CSS, JavaScript (без фреймворков)
- **Бэкенд**: PHP
- **База данных**: MySQL

### Структура проекта

- `public/` — публичная часть сайта (страницы и статические ресурсы)
  - `index.php` — главная страница
  - `services.php` — услуги
  - `about.php` — о компании и юристах
  - `contacts.php` — контакты
  - `login.php`, `register.php` — аутентификация
  - `client-dashboard.php` — личный кабинет клиента
  - `consultant-dashboard.php` — кабинет юриста/админа
  - `chat.php` — интерфейс онлайн‑консультанта
  - `assets/css/` — стили
  - `assets/js/` — скрипты
  - `assets/images/` — изображения и иконки
- `backend/` — серверная логика на PHP
  - `config/db.php` — параметры подключения к MySQL
  - `lib/` — переиспользуемые функции (аутентификация, логика бота и т.п.)
  - `handlers/` — обработчики форм и AJAX‑запросов
- `db/`
  - `schema.sql` — схема БД (DDL)
  - `seed.sql` — тестовые данные (DML)
- `docs/`
  - `ER-diagram.*` — ER‑диаграмма БД

### Установка и запуск (локально, под XAMPP)

1. **Скопировать проект в XAMPP**
   - Склонировать/скопировать папку проекта в каталог `htdocs`, например:
     - `C:\xampp\htdocs\yurkrass`

2. **Создать БД**
   - Запустить XAMPP (Apache + MySQL).
   - Открыть `phpMyAdmin` по адресу `http://localhost/phpmyadmin/`.
   - Создать базу данных, например `yurkrass_db` (utf8mb4).
   - Выполнить SQL‑скрипт из файла `db/schema.sql`.
   - По желанию — выполнить `db/seed.sql` для заполнения тестовыми данными.

3. **Настроить подключение к БД**
   - В файле `backend/config/db.php` указать:
     - хост (обычно `localhost`),
     - пользователя (обычно `root`),
     - пароль (по умолчанию пустой в XAMPP, если вы его не меняли),
     - название БД (`yurkrass_db` или другое выбранное имя).

4. **Открыть сайт**
   - В браузере перейти по адресу:
     - `http://localhost/yurkrass/public/index.php`

### Краткое описание функционала

- **Информационные разделы**:
  - главная страница с описанием компании «Юркрас» и преимуществами;
  - список юридических услуг с описанием и ценами «от»;
  - страница «О компании / Юристы» с краткой информацией о специалистах;
  - страница «Контакты» с формой обратной связи.

- **Онлайн‑консультант**:
  - чат‑бот, который на основе базы знаний в БД подбирает краткий ответ и предлагает ссылки на релевантные разделы (услуги, FAQ и т.п.);
  - возможность переключиться из режима бота в режим общения с живым консультантом;
  - история сообщений хранится в базе данных.

- **Личный кабинет клиента**:
  - просмотр личных данных (ФИО, дата рождения, телефон, email);
  - просмотр истории обращений и заявок на консультацию.

- **Кабинет юриста/админа**:
  - просмотр заявок на консультацию;
  - просмотр активных чат‑сессий и подключение к диалогу;
  - базовое управление услугами (создание/редактирование/отключение).

### Архитектура (как работает проект)

Проект построен по простой схеме «страницы → обработчики → база данных»:

- **Публичная часть (UI)**: папка `public/`
  - страницы (`index.php`, `services.php`, `about.php`, `contacts.php`, `chat.php`);
  - стили и JS: `public/assets/css/`, `public/assets/js/`;
  - общая шапка: `public/partials/header.php` (показывает кнопки входа/выхода в зависимости от сессии).

- **Серверная часть (логика на PHP)**: папка `backend/`
  - подключение к БД: `backend/config/db.php`;
  - общие функции:
    - `backend/lib/auth.php` — регистрация/вход/выход и проверка авторизации,
    - `backend/lib/chat_logic.php` — подбор ответа чат‑бота из базы знаний;
  - обработчики форм и AJAX:
    - `backend/handlers/handle_register.php` — регистрация клиента,
    - `backend/handlers/handle_login.php` — вход,
    - `backend/handlers/handle_logout.php` — выход,
    - `backend/handlers/handle_consultation_request.php` — заявка с формы «Контакты»,
    - `backend/handlers/handle_chat_message.php` — сообщения онлайн‑чата (AJAX).

- **База данных (MySQL)**: схема в `db/schema.sql`
  - хранит пользователей, услуги, заявки, чаты и базу знаний бота.

Ключевые сценарии:

- **Регистрация/вход**:
  - `public/register.php` → `backend/handlers/handle_register.php` → таблица `users`
  - `public/login.php` → `backend/handlers/handle_login.php` → таблица `users` → создаётся сессия (`$_SESSION`)

- **Заявка на консультацию**:
  - `public/contacts.php` → `backend/handlers/handle_consultation_request.php` → таблица `consultation_requests`

- **Онлайн‑консультант**:
  - `public/chat.php` + `public/assets/js/chat.js` → `backend/handlers/handle_chat_message.php`
  - `handle_chat_message.php` создаёт `chat_sessions`, записывает `chat_messages`, затем получает ответ через `backend/lib/chat_logic.php` (поиск в `bot_knowledge`).

### ER‑диаграмма БД (словесное описание)

База данных содержит основные таблицы:

- `users` — пользователи системы (клиенты, консультанты, администраторы);
- `services` — юридические услуги;
- `consultation_requests` — заявки на консультацию;
- `chat_sessions` — сессии чата (диалоги);
- `chat_messages` — сообщения в рамках сессий чата;
- `bot_knowledge` — база знаний для чат‑бота (шаблоны вопросов и ответы).

Связи (основные):

- один пользователь может иметь много заявок на консультацию (`users` 1 — M `consultation_requests`);
- одна услуга может фигурировать во многих заявках (`services` 1 — M `consultation_requests`);
- один клиент может иметь много сессий чата (`users` 1 — M `chat_sessions`);
- одна сессия чата содержит много сообщений (`chat_sessions` 1 — M `chat_messages`);
- один консультант (пользователь) может отправлять много сообщений (`users` 1 — M `chat_messages` по `sender_id`);
- одна услуга может быть связана с несколькими записями в базе знаний бота (`services` 1 — M `bot_knowledge`).

Графическое представление ER‑диаграммы прилагается в папке `docs/` (файл `ER-diagram.*`).

### Источники иконок и изображений

- Иконки (телефон, почта, пользователь, юридическая тематика) — SVG‑файлы, скачанные из набора **Font Awesome Free** или аналогичных бесплатных наборов, сохранённые локально в папке `public/assets/images/icons/`.
- Фотографии юристов/офиса могут быть добавлены позже в папку `public/assets/images/photos/` и подключены вместо заглушек.

### Версия для слабовидящих (план)

Проект предусматривает отдельный CSS‑файл с упрощённым оформлением (минимум цветов и графики, повышенный контраст и крупный шрифт). Переключение режима будет выполняться на уровне фронтенда (кнопка/переключатель, активация `accessible.css`).
